## 1. プロダクト概要

### 1.1 プロダクト名

**GeoScope**（仮称）

### 1.2 ビジョン

「開いたら見える、触れる、コードになる」

地球惑星流体データを、専門知識やスクリプト記述なしに即座に可視化・探索でき、
GUI操作がそのまま再現可能なコードとして記録されるデスクトップアプリケーション。

### 1.3 ポジショニング

GFD（地球流体力学）研究における可視化ツールの世代交代を目指す。
GrADS・NCL・Panoply が担ってきた役割を、現代的な UX で再定義する。

### 1.4 dcmodel との関係

dcmodel ファミリー（ispack-rs, spmodel-rs 等）の出力可視化を主要ユースケースとするが、
CF Conventions 準拠の NetCDF データ全般を対象とする汎用ツールとして設計する。
fludarium（流体可視化デモ）とは独立したプロダクトとする。

### 1.5 fludarium との棲み分け

| | **fludarium** | **GeoScope** |
|---|---|---|
| 用途 | リアルタイムシミュレーション可視化 | ポスト処理・解析可視化 |
| 入力 | パイプ (`--stream`), ライブデータ | ファイル (NetCDF) |
| 対象 | デモ、にぎやかし | 研究・教育 |
| UI | ミニマル（キーバインド中心） | フルGUI（パネル、Inspector 等） |

fludarium の球面レンダリング・カラーマップ等の実装は GeoScope に移植・再利用する。

## 3. デザイン原則

| # | 原則 | 説明 |
|---|------|------|
| 1 | **Zero-config first view** | ファイルを開いたら 0 秒で「賢い初期描画」を表示する |

## 4. 機能要件

### 4.1 ファイル読み込みと自動描画

**概要**: NetCDF ファイルを開いた瞬間に、メタデータから最適な描画法を自動推論して表示する。

**自動推論ロジック（3段階フォールバック）**:

> **重要**: dcmodel (gtool-rs) の出力はフィールド変数に CF `standard_name` を持たない。
> Level 1 だけに依存せず、3段階フォールバックを必須とする。

**Level 1 — CF standard_name（確信度: 高）**:
- CF Conventions の `standard_name` / `units` から変数の種類を判定
- 該当すれば最適な描画法・カラーマップを自動選択

**Level 2 — ヒューリスティクス（確信度: 中）**:
- 変数名パターンマッチ: `vor*` → 渦度, `temp*`/`T` → 温度, `d*` → 偏差(diverging)
- `long_name` 属性からの推論
- `units` 属性: `K` → 温度, `m/s` → 速度, `Pa` → 気圧
- gtool-rs 独自属性の活用: `grid_type`, `truncation_nm`, `model`, `params`
- 座標変数名の柔軟認識: `lon`/`longitude`/`x`/`g0_lon_1`/`nav_lon` 等

**Level 3 — 次元構造のみ（確信度: 低、最終フォールバック）**:
- 2D (`dim1 x dim2`) → カラーフィル + viridis
- 1D → ラインプロット
- データ統計による diverging/sequential 自動判定: `min * max < 0` → diverging (RdBu_r), else → sequential (viridis)

**描画法の自動選択**:
- 座標軸 (`lon`, `lat`, `lev`, `time`) を自動認識
- 2D (`lon x lat`) → Globe view + カラーフィル
- 3D (`lon x lat x lev`) → Globe view（地表レベル）+ 鉛直スライダー追加
- 時系列あり → タイムスライダー追加

**カラーマップ自動選択**:

| 変数 | カラーマップ | タイプ |
|------|------------|--------|
| 温度（絶対値） | inferno / thermal | sequential（常に正） |
| 温度偏差 / アノマリ | RdBu_r | diverging, 0中心 |
| 渦度 | RdBu_r | diverging, 0中心 |
| 発散 D | RdBu_r | diverging, 0中心 |
| 帯状風 u | RdBu_r | diverging, 0中心（東風-/西風+） |
| 降水量 | YlGnBu | sequential |
| ジオポテンシャル | viridis | sequential |
| 地形 | terrain | sequential |
| `d*` プレフィックス (dvort, dphi) | RdBu_r | diverging（自動偏差検出） |
| 不明 + min*max < 0 | RdBu_r | diverging |
| 不明 + min*max >= 0 | viridis | sequential |

**推論結果の透明性**（原則 5: Teach, don't hide）:
- Inspector に「"渦度" と推論しました（変数名 `vor` から判定）」のようなヒントを表示
- ワンクリックで変数タイプを修正可能
- 学習機能: ユーザーが修正したパターンを記憶し、次回以降に適用

**入力方法**:
- ドラッグ & ドロップ
- メニューからファイル選択
- ファイル関連付けによる起動
- コマンドラインからの起動 (`geoscope open file.nc`)

### 4.2 球面操作（Direct Manipulation）

**マウス操作**（Google Earth / Cesium 標準に準拠）:

| 操作 | 動作 |
|------|------|
| 左ドラッグ | 地球回転 |
| 右ドラッグ | パン（視点移動） |
| スクロール | ズーム（領域拡大、自動投影法遷移） |
| クリック | データ依存: 時間軸あり→時系列、なし→値表示、鉛直軸あり→プロファイル |
| Shift+クリック | ピンを立てる（複数点の比較） |

**断面操作（モード切替方式）**:
- ツールバーのハサミアイコンをクリック → 断面モードに入る
- 断面モード中: 左ドラッグで断面線を引く → Section ビューが自動で開く
- `S` キー押下中にドラッグでも断面モードに入れる（上級者向け）

**トラックパッド / タッチジェスチャー** (macOS):

| 操作 | 動作 |
|------|------|
| 1本指ドラッグ | 地球回転 |
| 2本指スクロール | ズーム |
| ピンチ | ズーム |

**キーボードショートカット**:

| キー | 動作 |
|------|------|
| Space | 時間アニメーション再生/停止 |
| ← → | 時間ステップ移動 |
| 1, 2, 3, 4, 5, 6 | ビュー切替（Globe / Map / Section / Profile / Hovmoller / Spectrum） |
| S + ドラッグ | 断面モード |
| Cmd+Z / Ctrl+Z | アンドゥ |

### 4.3 ビュータイプ

| ビュー | 説明 | 優先度 |
|--------|------|--------|
| **Globe** | 正射影の 3D 地球儀。回転・ズーム可能 | P0 |
| **Map** | 2D 地図投影。デフォルト: Equirectangular（GrADS 標準） | P0 |
| **Hovmoller** | ホフメラー図（時間-経度 / 時間-緯度）。波の伝播方向・速度を一目で把握 | P0 |
| **Spectrum** | E(n) エネルギースペクトル図（1D log-log）。n^{-3} 参照線付き | P0 |
| **Section** | 鉛直断面図（経度固定 / 緯度固定 / 任意断面）。Zonal Mean トグル付き | P1 |
| **Profile** | 特定地点の鉛直プロファイル / 時系列 | P1 |

**Map ビューの投影法**:

| 投影法 | 説明 | 優先度 |
|--------|------|--------|
| Equirectangular (Plate Carree) | **デフォルト**。GrADS ユーザーの移行パス | P0 |
| Mollweide | 全球の面積保存表示。論文でよく使われる | P0 |
| Robinson | 全球表示。見た目のバランスが良い | P1 |
| Polar Stereographic | 極域研究。極渦の可視化に必須 | P1 |
| Lambert Conformal Conic | 中緯度の局所拡大 | P2 |

### 4.4 ベクトル場の重ね合わせ

スカラーフィールド上にベクトル場（風向風速等）を矢印またはストリームラインで重ね描き。
spmodel-rs は `u_cos`, `v_cos`（cos phi で重み付けされた速度場）を出力する。

| 描画法 | 説明 | 優先度 |
|--------|------|--------|
| Wind arrows | 等間隔の矢印。向き=風向、長さ=風速 | P0 |
| Streamlines | 流線。流れの全体像を把握 | P1 |
| Wind barbs | 気象学標準の記法 | P2 |

### 4.5 Inspector パネル

GUI で調整可能な描画パラメータ:

- 変数選択（ドロップダウン）
- 投影法選択
- カラーマップ選択（プレビュー付き）
- 値範囲（自動 / 手動）
- 等値線レベル（自動 / 手動）
- 時間ステップ / 鉛直レベル
- ポイント情報（カーソル位置の座標・値）

### 4.6 コード双方向同期

**同期方式: GUI マスター方式**

Code Panel は「現在の GUI 状態のスナップショット」を常に反映する。
コードと GUI が矛盾する問題を構造的に回避する。

- GUI 操作 → Code Panel が自動更新（デバウンス: 操作静止 0.5秒後に確定）
- ユーザーがコードを編集中 → 自動更新を一時停止
- 「実行」ボタン押下 → コードが GUI 状態を上書き
- 球面回転中は `center=(lon, lat)` の値のみ連続更新し、操作完了後に1行として確定

**GUI → コード**:
- すべての GUI 操作が意味レベルのコードとして Code Panel に表示される
- 生成コードは Python (xarray / matplotlib 互換) を第一候補とする

```python
# GeoScope auto-generated
import geoscope as gs

ds = gs.open("exp01.nc")
fig = ds.vorticity.plot(
    time="48h",
    projection="orthographic",
    center=(135, 35),
    cmap="RdBu_r",
    symmetric=True,
)
fig.save("vorticity_48h.png")
```

**コード → GUI**:
- Code Panel でコードを編集 →「実行」ボタンで Viewport に反映
- v0.2 では Python テキスト生成のみ（実行は外部 Python で行う前提）
- 将来的に Rhai / PyO3 による組み込み実行を追加

**レシピ保存**:
- 操作をスクリプトファイル (.py) として保存
- `geoscope run recipe.py` で同じ図を再現
- 教員が学生に配布するユースケースを想定

### 4.7 Progressive Disclosure（段階的 UI 開示）

**Level 0 — 初回起動**:
- ウェルカム画面: ファイルドロップエリア + サンプルデータで試すボタン
- Data Browser + Viewport のみ表示

**Level 1 — 基本操作**:
- Data Browser + Viewport + Time Slider
- Inspector は折りたたみ（右端にティーザータブ「設定」が見えている）
- Code Panel は非表示（下端にプレビュー1行 `>>> # コードを見る` が見えている）

**Level 2 — 探索モード**:
- Inspector を展開 → 投影法、カラーマップなどを調整
- 比較モード ON → パネル分割

**Level 3 — コードモード**:
- Code Panel を展開 → GUI 操作がコードとして表示
- コード編集 → 描画に反映
- スクリプトとして保存

**遷移トリガー**:

| 遷移 | トリガー | 方式 |
|------|---------|------|
| L0 → L1 | ファイルを開いた瞬間 | **自動** |
| L1 → L2 | Inspector 折りたたみクリック / カラーマップ変更試行 | **手動（ナッジあり）** |
| L2 → L3 | Code Panel 展開 / Cmd+K コマンドパレット | **手動** |

**ナッジ（誘導）の手法**:
- パネル端のティーザー: 折りたたみ時も右端/下端にタブが見えている
- コンテキスト駆動ヒント: 同じ操作を3回繰り返す → 「コードで自動化できます」
- **ポップアップ/モーダルは使わない** — 研究者の集中を妨げない

### 4.8 データブラウザ

- 開いたファイルの変数をツリー表示
- 変数のメタデータ（単位、次元、サイズ）をツールチップ表示
- 変数をダブルクリック → 即座に描画
- 複数ファイルを同時に開ける

## 7. 技術スタック

| レイヤー | 技術選定 | 選定理由 |
|---------|---------|---------|
| GUI フレームワーク | **egui + egui_dock** | ドッキング UI 成熟、wgpu 公式統合。Iced はドッキング不在で不採用 |
| 3D レンダリング | **wgpu** | WebGPU ベース、将来ブラウザ移植可能。60fps 余裕 |
| NetCDF 読み込み | **netcdf-rs** (v0.12.0) | georust 傘下でアクティブメンテ。libnetcdf の C 依存あり |
| 地図投影 | 自前実装 | ispack-rs の球面座標変換知見を活用 |
| コード生成 | **Python テキスト生成** | xarray / matplotlib 互換コードを文字列として出力（PyO3 不要） |
| スクリプトエンジン (MVP) | **Rhai** | 軽量・安全・配布問題なし。バイナリ組み込み |
| スクリプトエンジン (将来) | **PyO3** | 研究者に馴染む Python 環境。配布コストが課題なので後回し |

### Globe 描画アーキテクチャ

```
[NetCDF data] → [CPU: 2D array] → [GPU: 2D Texture (lon x lat)]
                                          |
[Colormap LUT] → [GPU: 1D Texture] → [Fragment Shader: lookup]
                                          |
                                   [Sphere Mesh → Screen]
```

- カラーマップ変更 = 1D テクスチャ差し替えのみ
- データ更新 = 2D テクスチャ転送のみ
- fludarium の既存レンダラーを移植・再利用

### ピボット判断

v0.1a 完了時点 (2ヶ月) で egui + wgpu の統合品質を評価する。
Inspector / Data Browser の実装が困難な場合、**Tauri (Web フロント)** へのピボットを検討する。

**将来のブラウザ対応**: wgpu は WebAssembly + WebGPU へのコンパイルが可能。
デスクトップファーストで設計しつつ、将来のブラウザ版への道を閉じない。

## 付録 A: 既存ツールの課題分析

### A.1 「コードを書かないと何も見えない」問題

GrADS, NCL, Matplotlib 系はすべて「まずスクリプトを書く」前提。
データを開いて「ざっと見る」だけで 10〜20 行のコードが必要。
地図投影の設定、カラーマップ、等値線レベルなど「お約束」の設定が毎回必要。
探索的解析のフリクションが極めて高い。

### A.2 「時代が止まった GUI」

Panoply, GrADS の GUI は 2000 年代の Java Swing / Tcl/Tk 感覚。
高 DPI ディスプレイ非対応、D&D なし、アンドゥなし、ダークモードなし。

### A.3 「球面データ特有の操作」が貧弱

地球を回転させながらデータを見る操作が直感的でない。
断面図の切り替えが手間。アニメーションのシークバーがない or 貧弱。
スペクトル空間と格子点空間の行き来を可視化できるツールがほぼない。

### A.4 「大規模データへのスケーラビリティ」

T213 (640x320) 程度でも時系列を持つと数十 GB。
全球雲解像モデル（数 km 格子）は TB 級。メモリ不足で固まる。

### A.5 「比較」が難しい

2 つの実験結果を並べて見るのが面倒。差分のインタラクティブ表示がない。
観測データとモデル出力の overlay が困難。

### A.6 「共有・再現性」の断絶

可視化の状態を保存・共有する仕組みがない。
論文用の publication-quality 図とインタラクティブ探索が完全に別ワークフロー。

### A.7 学習曲線の壁

各ツールの独自スクリプト言語の習得コストが高い。
「先輩のスクリプトをコピーして改変」文化による理解なき利用。

## 付録 B: 参考ツール一覧

### 研究コミュニティの定番

- **GrADS**: COLA/IGES, 1988年〜。CLI + スクリプト
- **NCL**: NCAR, 2019年メンテナンスモード。スクリプト言語
- **ParaView**: Kitware/Sandia。GUI (Qt)、CFD 全般
- **Panoply**: NASA GISS。GUI (Java/Swing)、NetCDF/HDF 簡易閲覧
- **VAPOR**: NCAR。GUI (Qt + OpenGL)、3D 大気・海洋データ
- **Ferret / PyFerret**: NOAA PMEL。CLI + スクリプト、海洋データ

### Python エコシステム

- **Matplotlib + Cartopy**: 2D 地図投影の定番
- **xarray + hvPlot**: NetCDF 直接扱い + インタラクティブ
- **Plotly / Dash**: Web ベースダッシュボード
- **PyVista**: 3D 可視化 (VTK ラッパー)
- **UXarray**: 非構造格子データ向け

### Web ベース

- **Cesium**: 3D 地球儀 WebGL
- **deck.gl**: 大規模地理空間可視化
- **NASA Worldview**: 衛星データ Web 閲覧

### GFD 電脳倶楽部系

- **gtool**: 独自バイナリフォーマット + 可視化
- **DCL**: Fortran 描画ライブラリ
- **gpview / figgen**: DCL ベースの簡易可視化コマンド
